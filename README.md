# WTA-Homework-Coach-Skill

帮你系统化完成课程作业:
少点纠结的同时，保留个人味道，减少AI味。

---

## 项目背景

本 Skill 为完成 **WaytoAGI 线上 AI 训练营** 作业而开发。目前版本针对第4期训练营开发调优。训练营有多课作业（痛点挖掘、MVP 架构、全栈实现、AI Native、个人主页等），用它可以按步骤推进、保证提交物质量，省去反复纠结。
> 请务必人工审核所有AI生成的内容，因为您始终需要对AI生成的内容负责；禁止未经审阅直接使用AI生成的内容提交作业！

## 你能得到什么

- **有节奏地推进**：启动 → 规划 → 写作 → 审查 → 配图，每一步都有明确动作。
- **作业像你写的**：会先引导你输入想法和细节，再动笔，减少泛泛的「AI 腔」。
- **文风可控**：支持用你的笔记或历史文章做文风参考，输出更贴近你的表达习惯。
- **可选配图**：定稿后可按文档中的占位一键批量出图。

## 适合谁 / 不适合谁

**适合：**

- 参加 **WaytoAGI 训练营** 或其他 AI/产品类课程、有几课作业要交的同学。
- 希望 AI 帮忙规划与起草，但不想交出一份「一眼 AI」的作业。
- 愿意在动笔前花几分钟想清楚：核心观点、真实感受、关键细节。

**不适合：**

- 想要一键成稿、不愿准备资料或回答规划问题，想要应付了事作业的同学。
- 完全不想碰配置文件或目录结构的同学（需简单填写路径与课次映射）。

## 重要：先输入你的想法，再生成内容

Skill 会**主动引导你输入**这些内容，然后才开始写正文：

- 你这课最想表达的核心观点  
- 真实感受：哪里最痛、最卡、最想吐槽  
- 关键细节：具体日期、场景、动作  
- 边界要求：哪些能写、哪些不能写  
- 交稿偏好：更叙事还是更结构化  
- 作品北极星：你希望读者记住的一句或一段话  

**没有这些，不会开始写正文。** 这样写出来的作业才像你的，而不是模板填空。

## 使用前要准备什么

### 你要准备的材料

| 类型 | 放在哪（配置里的「输入目录」） | 示例 |
| --- | --- | --- |
| 作业题目、建议、草稿 | 输入目录（ `ai-learn/OtherINPUT/`） | 作业要求.md、课程回放文稿、讲义 |
| 文风参考（可选） | 文风参考目录（ `ai-learn/HistoricalArticles/`） | 你的笔记或历史本人写作文章，挑选2-3篇能代表你个人风格的文章，可以提高文风相似度 |
| 产品/项目资料（做产品课时） | 由课次映射指向的项目目录 | PRD、TODO、代码仓库等 |

**没有这些资料，Skill 无法正确读取作业要求和文风，会提示缺失。** 使用前请先准备好。

### 你要有的环境

- **Cursor** 或 **Claude Code**（任选其一）。
- 愿意填两份配置：作业根路径、课次与必读素材映射（见下方「安装与配置」）。
- **模型能力**：尽量使用高品质模型。最低不低于 Claude Sonnet 4.5；推荐 Google Gemini 3.1 / GPT 5.2；复杂工程建议使用 Claude Opus 4.6 或 GPT 5.3 Codex Extra High 或更强模型。

## 使用时会经历什么

你说「开始第 1 课作业」→ AI 先检查配置、读你的资料 → **问你几个问题**（级别、核心观点、感受、细节、作品北极星等）→ 你回答 → AI 出一份**写作计划**给你确认 → 你确认后才动笔 → 写完再一起审查、定稿 → 需要的话再配图。

| 阶段 | 你会经历什么 | 你要做的 |
| --- | --- | --- |
| 00 启动 | AI 检查配置、加载进度、读取你放的资料 | 缺配置就按提示补全 |
| 01 规划 | AI 问你：选哪种级别、核心观点、真实感受、关键细节、边界、偏好、「作品北极星」 | 认真回答，原话会写进作业 |
| 02 写作 | AI 按计划写出初稿 | 确认计划后再动笔 |
| 03 审查 | AI 做后处理、质量审查，把稿子交给你审阅 | 满意再定稿 |
| 04 配图（可选） | AI 根据文档里的占位标记批量出图 | 定稿后按需执行 |

## 安装与配置

### 1. 放置 Skill

本仓库根目录就是 skill 内容（没有 `.cursor` 或 `.claude` 这类父级文件夹）。把**整个仓库根目录下的全部内容**复制到：

- **Cursor**：`你的项目/.cursor/skills/homework-coach/`
- **Claude Code**：`你的项目/.claude/skills/homework-coach/`

复制后，`SKILL.md` 应该在 `homework-coach/SKILL.md` 这个位置。

### 2. 填写必填配置

在 skill 目录下：

- 编辑 `configs/project-homework.md`：填你的**作业根目录**路径，以及下面的输入目录、文风目录、输出目录（或用默认结构）。
- 编辑 `configs/lesson-map.md`：填**课次和每课要读的素材**（作业要求、笔记、项目路径等）。

若 `lesson-map.md` 里用了项目别名（如 `AB/`），再在 `configs/` 下增加对应的 `project-ab.md`（可从 `configs/project-template.example.md` 复制）。

### 3. 把资料放到约定位置

路径在 `configs/project-homework.md` 中配置。若沿用模板中的默认结构，则：**IN/** 对应作业根下的输入目录（如 `HW/ai-learn/OtherINPUT`）→ 放**作业题目、建议、草稿**；**STYLE/** 对应文风参考目录（如 `HW/ai-learn/HistoricalArticles`）→ 放**文风参考**；**OUT/** 为输出目录（如 `HW/ai-output`），由 Skill 写入，无需事先放资料。请将作业题目、建议、草稿放到 IN 对应目录，文风参考放到 STYLE 对应目录；若已自定义路径，则放到你在配置中填写的目录。

### 4. 触发 Skill

在 Cursor 或 Claude Code 里说一句「开始第 1 课作业」或「帮我做第 2 课作业」。阶段 00 会先自检配置，缺什么会提示你怎么补。

## 可选：配图（生图 API）

配图对应流程中的**阶段 04**，在审查定稿之后进行；是否执行可选，不影响前序阶段。若要用定稿后的批量出图：

1. 在 skill 目录下复制 `scripts/apimart.env.example` 为 `scripts/apimart.env`，填入密钥（`TOKEN` 或 `SK` 或 `API_KEY`）。**不要将 `apimart.env` 提交到版本控制。**
2. 准备 `requests.jsonl`（每行一张图的请求）。
3. 运行脚本时使用**真实路径**（不要用 `OUT/` 这类别名）。

详见 `scripts/README.md`。

## 常见问题（你说人话，我对得上）

- **「一打开就报错 / 说缺配置」**  
  检查是否已填写 `configs/project-homework.md` 和 `configs/lesson-map.md`；若没有，从 `configs/*.blank.md` 复制一份再填。

- **「AI 说找不到资料」**  
  确认作业题目、建议、草稿等已放到你在配置里写的「输入目录」和「文风参考目录」。

- **「写出来的不像我」**  
  规划阶段（01）里 AI 会问你核心观点、感受、细节、作品北极星；认真回答，原话会进正文。若跳过了或答得很空，就容易像模板。

- **「提示别名缺少 project-xxx.md」**  
  在 `lesson-map.md` 里用了某个项目别名（如 `AB/`），就要在 `configs/` 下有一个 `project-ab.md`，可从 `configs/project-template.example.md` 复制后改。
