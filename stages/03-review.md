# 阶段 03：审查与交付

**目标：** 对初稿执行强制后处理和质量审查，交付给用户审阅，根据反馈迭代直到通过。

---

## 步骤

### 1. 执行强制后处理（7 项扫描）

这不是可选步骤。每次写完作业正文，交稿前必须逐项扫描并修正：

1. **日期扫描**：查找所有 `YYYY-MM-DD` 格式（如 `2026-01-11`），替换为 `YYYY年M月D日`（如 `2026年1月11日`）。漏一个就是违规。
2. **空格扫描**：查找所有「数字 + 空格 + 中文量词/名词」（如 `19 天`、`8 份`、`50 分钟`），删掉空格改为 `19天`、`8份`、`50分钟`。
3. **列表扫描**：查找正文叙事段中的 `- **XX**：` 模式，全部改写为段落叙述。
4. **句式扫描**：查找「数据前置」句式（日期/数字出现在句首主语位），改为「意义前置 + 数据括号补充」。
5. **元评论扫描**：查找任何关于"我会改""如果你觉得""下一步我打算"之类的自我评论，全部删掉。
6. **段落长度扫描**：查找超过 5 句话的段落，拆分为 3~5 句的短段；特别注意"首先…其次…最后…"结构，每个子论点必须独立成段。
7. **平庸句扫描**：查找空泛套话（如"随着时代发展""具有重要意义""值得深入思考"），改成用户自己的具体观点、感受与场景细节。

**以上 7 项全部完成后才允许输出。如果用户看到任何一项违规，说明后处理没做。**

### 2. 质量审查

审查时可以（也应该）：
- 按 `configs/lesson-map.md` 当前课次涉及的别名，读取对应 `configs/project-*.md` 指向的项目代码并审查
- 示例：若课次素材含 `WW/`、`OP/`，则分别读取其对应项目配置
- 读取 `lesson-map` 中配置的讲义/背景资料路径，确认作业要求解读准确

审查维度：

| 维度 | 检查内容 |
| --- | --- |
| 文风一致性 | 输出是否与用户文风参考一致？列表是否超限（30%）？叙事段是否被拆成碎片列表？ |
| 事实核查 | 接口、功能、MVP 边界是否与项目源文档一致？ |
| AI 味检测 | 逐段扫描是否有"绝对禁止"列表中的模式 |
| 拆分检查 | 提交物 A 和 B 是否真的独立？改 A 会不会牵动 B？ |
| 出处检查 | 引用的项目概念是否标注了来源？ |
| 格式检查 | 正文叙述段是否用了完整段落？每段是否至少 3 句话？ |
| 图示密度 | 每 800 字正文是否至少有 1 个 IMAGE 占位？封面图、核心论点图、循环/机制图是否都有？ |
| 图示质量 | Mermaid 图是否正确渲染？IMAGE 占位标记描述是否具体到可直接生成提示词？ |
| 作品级门槛 | 读完后是否能记住用户的一句话立场？是否存在“能交但不出彩”的段落？ |

### 3. 交付用户审阅

【确认点②】将审查通过的稿件交付给用户。

- 展示稿件内容（或告知输出文件位置）
- 简述审查结果
- 等待用户反馈

### 4. 迭代修改

- 用户反馈 → 修改 → 重新执行后处理 → 再交付
- 循环直到用户确认通过
- 修改时保持其他部分不变，只改用户指出的问题

### 5. 收尾

用户确认通过后：
- 提醒用户手动 Git 提交/部署（不代替执行）
- 更新 `HW/homeworkTODO.md` 中的步骤 checkbox
- 如果是课级状态变更，告知用户并等待确认后更新 `HW/homeworkTODO.md` 中的课级状态
- **如果文档中有 IMAGE 占位标记**，告知用户可以进入阶段 04 生成配图

## 交稿前自检清单

> 完整版见 `templates/checklist.md`

- [ ] **读了没？** 是否已读取对应课次的全部必读源文档？
- [ ] **学了没？** 是否已读取文风参考文件并提炼了特征清单？
- [ ] **拆了没？** 多级别作业是否已拆分为独立提交物？
- [ ] **对了没？** 产品概念/接口名/MVP 边界是否与源文档一致？
- [ ] **标了没？** 引用项目文档的地方是否标注了出处？
- [ ] **扫了没？** 是否完成了 7 项强制后处理？
- [ ] **闭嘴没？** 输出里是否有元评论、自我预告、风格讨价还价？有就删掉
- [ ] **问了没？** 关键决策点是否暂停等待用户确认了？
- [ ] **图够没？** IMAGE 密度是否达标（每 800 字至少 1 个）？封面图、核心论点图、循环图是否都有？
- [ ] **图好没？** Mermaid 图能渲染？IMAGE 描述具体到可直接出提示词？
- [ ] **顶不顶？** 是否达到了“可以代表用户水平”的作品线？若没有，继续重写薄弱段落
