# 阶段 00：启动与上下文加载

**目标：** 加载进度状态、检测未转换文件、读取必要源文档，让 AI 快速进入工作状态。

---

## 步骤

### 1. 环境与规则检查（最优先）

**检查工作区规则文件是否就绪。** 这一步在所有其他步骤之前执行。

#### 检测逻辑

```
读取工作区根目录的 .cursorrules
      │
      ├─ 文件存在且有内容 → 通过，继续下一步
      │
      ├─ 文件存在但为空 / 文件不存在 → 检查 .cursor/ 目录是否存在
      │     │
      │     ├─ .cursor/ 存在（= Cursor 环境）
      │     │     → 读取 templates/cursorrules-default.md 中的模板内容
      │     │     → 创建 .cursorrules 并写入默认规则
      │     │     → 告知用户："已创建 .cursorrules，请检查路径缩写是否正确"
      │     │
      │     └─ .cursor/ 不存在（= 非 Cursor 环境）
      │           → 提示用户：
      │             "检测到你可能不在 Cursor 中运行。
      │              请手动将以下规则添加到你的 IDE 工作区规则配置中。"
      │           → 输出 templates/cursorrules-default.md 中的规则内容
      │           → 附上各 IDE 对应的规则文件位置表
      │           → 等待用户确认已添加后再继续
      │
      └─ 读取失败（权限等问题）→ 告知用户手动检查
```

#### 关键约束

- 创建 `.cursorrules` 后，提醒用户确认路径缩写中的**绝对路径**是否匹配当前机器
- `.cursorrules` 创建后即为只读——后续步骤不得再修改它
- 如果用户的 IDE 不支持自动加载规则文件，AI 应在每次对话开头手动遵循 `templates/style-rules.md` 中的写作铁律

### 2. 配置完整性自检（复用必做，先于读取任何项目路径）

在读取任何项目文件前，先做“核心配置 + 按需配置”自检。

**核心必检（固定 2 个）：**

1. `configs/project-homework.md`
2. `configs/lesson-map.md`

**按需必检（动态）：**

- 读取 `configs/lesson-map.md`，提取其中出现的项目别名（表格与通用必读里出现的 `XXX/` 形式，排除 `HW/`、`IN/`、`STYLE/`、`OUT/`）
- 对每个别名，检查是否存在对应项目配置文件：`configs/project-<别名小写>.md`（如 `AB/` -> `configs/project-ab.md`）
- 若某别名缺少对应 `project-*.md`，列出缺失项并提示用户从 `configs/project-template.example.md` 复制为 `configs/project-<别名小写>.md` 后填写

#### 自检失败处理

- **核心文件缺失**：立即暂停，输出以下引导，等待用户补全后继续：
  - 缺 `project-homework.md`：请将 `configs/project-homework.blank.md` 复制为 `configs/project-homework.md`，填写根路径与 `HW/IN/STYLE/OUT` 别名。
  - 缺 `lesson-map.md`：请将 `configs/lesson-map.blank.md` 复制为 `configs/lesson-map.md`，填写课次与必读素材。
- **动态项目配置缺失**：立即暂停，列出 `lesson-map.md` 中出现的别名及建议文件名（规则：别名 `XY/` -> `configs/project-xy.md`），提示用户从 `configs/project-template.example.md` 复制并填写。
- **任一文件为空**：立即暂停，告知“配置文件为空”，要求按上两条补全。
- 全部通过后，才允许进入下一步。

### 3. 加载项目路径配置（禁止硬编码）

在读取任何项目文件前，必须先读取以下配置文件：

1. `configs/project-homework.md`
2. `configs/lesson-map.md`（课次映射）
3. `lesson-map.md` 中实际用到的项目配置文件（动态加载）

约束：
- 仅通过配置文件解析 `HW/`、`IN/`、`STYLE/`、`OUT/` 与课次实际用到的项目别名
- 流程文件中不得新增绝对路径
- 路径或课次映射变更只允许修改 `configs/*.md`

### 4. 检查并加载进度文件

1. 使用已加载的 `project-homework.md` 解析 `HW/` 别名
2. 检查 `HW/homeworkTODO.md` 是否存在，不存在则创建
3. 读取 `HW/homeworkTODO.md` —— 获取完整进度与当前焦点任务（这是进度的唯一来源）

### 5. 执行姿态校准（作品级）

在进入规划和写作前，先完成一次“执行姿态校准”并告知用户：

- 本次目标不是“写完”，而是“交出可以代表用户水平的作品”
- 任何“凑字数、套模板、空话拉长、泛泛总结”都视为失败
- 若关键素材不足，先提问补齐，不允许带着空洞信息硬写

### 6. 检测并转换 docx 文件

扫描 `IN/` 和 `STYLE/` 目录：

- 如果存在 `.docx` 文件但没有同名 `.md`，调用子模块 [docx-to-md.md](../docx-to-md.md) 执行转换
- 转换完成后邀请用户评估 `.md` 文件的完整性
- **用户确认完整性之前，不得使用转换后的文件作为素材**

### 7. 执行强制阅读（MUST READ BEFORE WRITE）

**铁律：在输出任何作业内容之前，必须先读取对应的项目源文档。不读不写，读了再写。**

#### 必读来源（配置驱动）

按以下顺序读取，不得跳过：

1. 读取 `configs/lesson-map.md`，定位“当前课次”的必读素材清单
2. 对清单中的每个路径，按别名解析后逐项读取
3. 若 `lesson-map` 里定义了“通用必读素材”（如作业要求、用户笔记、讲义），也必须读取

#### 缺失处理（复用场景关键）

- `lesson-map` 未定义当前课次：立即暂停，要求用户补充映射
- 某必读路径不存在：立即告知缺失路径，等待用户修正或替代路径
- 若课次无足够素材，禁止进入写作阶段

#### 阅读后的输出约束

- **所有产品概念、MVP 边界、接口名称、技术选型必须来自项目源文档**，不得自行编造
- 引用源文档时注明出处：`（来源：{文件名} {章节/标识}）`
- 如果源文档中没有相关内容，明确告诉用户"项目文档中未找到 X 的定义，需要你补充"

### 8. 向用户报告状态

告诉用户：
- 当前在哪（哪课、什么状态）
- 下一步是什么
- 预计需要什么（素材、确认、截图等）

## 进度文件维护协议

### HW/homeworkTODO.md（完整版）

- 位置：`HW/homeworkTODO.md`，每次 Skill 启动时检查是否存在，不存在则创建
- 包含每课的：作业级别选择、细分步骤、检查清单、提交物状态
- 每个步骤完成后**自动更新**对应 checkbox，事后告知用户
- 课级状态变更（如 "未启动→进行中" 或 "进行中→已完成"）需**用户确认**后更新

### .cursorrules（只读，不修改）

- `.cursorrules` 是纯规则文件，包含写作铁律和资源路径速查
- **AI 不得修改 `.cursorrules`**，所有进度变更只写入 `HW/homeworkTODO.md`

## 作业-项目映射（来自配置）

- 映射单一来源：`configs/lesson-map.md`
- 启动阶段只读取并执行映射，不在流程文件重复维护映射表
