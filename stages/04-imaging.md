# 阶段 04：配图生成

**目标：** 将定稿文档中的 `<!-- IMAGE: ... -->` 占位标记转化为实际图片。先生成提示词，用户确认后批量出图或手动出图。

**触发条件：** 用户确认作业定稿后，且文档中存在 IMAGE 占位标记。用户也可以随时说"给这段配个图"触发单张生成。

---

## 流程概览

```
扫描定稿中的 IMAGE 标记
       │
       ▼
为每个标记生成图片提示词（Prompt Pack）
       │
       ▼
展示提示词清单给用户确认
       │
       ▼
用户选择出图方式
       │
   ┌───┴───┐
   ▼       ▼
 A) 手动  B) 批量 API
   │       │
   │       ▼
   │   生成 JSONL → 执行脚本
   │       │
   └───┬───┘
       ▼
替换占位标记为图片引用
```

---

## 步骤

### 1. 扫描占位标记

读取定稿文档（`OUT/L{N}.md`），提取所有 `<!-- IMAGE: ... -->` 标记。

输出图片清单：

```
共发现 N 处配图需求：
- 图1：{描述}（位置：第 X 行，上下文：{所在小节标题}）
- 图2：{描述}（位置：第 Y 行，上下文：{所在小节标题}）
```

### 2. 生成图片提示词（Prompt Pack）

为每张图生成完整的提示词。每张提示词必须包含：

1. **风格块**：粘贴 `templates/image-prompt-style.md` 定义的风格基准
2. **内容描述**：基于占位标记的描述 + 上下文（从文档中提取该位置前后的内容）
3. **文字要求**：图上需要出现的文字（中文，逐字指定）
4. **排版约束**：画幅、留白、字号、对齐
5. **负面约束**：不要额外小字/乱码/英文字符

**每张提示词用一个独立代码块输出，便于复制。**

输出示例：

````
### 图1：{标题}

位置：`## {所在小节}` 下方
用途：{用在哪里看，如"作业文档内嵌图"或"PPT投影"}

```
[完整提示词内容]
```
````

### 3. 用户确认并选择出图方式

展示所有提示词后，询问用户：

- **A) 手动出图**：用户自行复制提示词到其他平台（如 ChatGPT、Midjourney）生成
- **B) 批量 API 出图**：自动生成 JSONL 请求文件并调用 `scripts/apimart_batch_generate.py`

### 4A. 手动出图流程

- 用户自行生成图片
- 用户将图片文件放入 `OUT/images/`
- AI 帮用户替换文档中的占位标记为图片引用

### 4B. 批量 API 出图流程

> 前提：用户已配置 `scripts/apimart.env`（参照 `templates/api-config.md`；密钥字段支持 `TOKEN`/`SK`/`API_KEY`）

**生成请求文件（JSONL）：**

输出到 `OUT/images/requests.jsonl`，每行一张图：

```jsonl
{"id":"01","prompt":"...","size":"16:9","n":1,"resolution":"2K","model":"gemini-3-pro-image-preview","pad_url":""}
```

**执行批量出图：**

```bash
python scripts/apimart_batch_generate.py \
  --config scripts/apimart.env \
  --input "<requests.jsonl 的实际路径>" \
  --out "<图片输出目录的实际路径>"
```

说明：
- `OUT/` 是文档中的逻辑别名，命令行不直接解析。
- 执行脚本时请使用真实路径（绝对路径或相对当前目录路径）。

**输出结果：**
- 图片文件保存在 `OUT/images/`
- `run.json` 记录请求和返回信息

### 5. 替换占位标记

将文档中的 `<!-- IMAGE: ... -->` 替换为实际图片引用：

```markdown
![{图片描述}](images/{filename})
```

---

## 单张配图（随时触发）

当用户在非定稿阶段说"给这段配个图"时：

1. 根据用户指定的内容生成提示词
2. 展示提示词给用户确认
3. 用户选择手动出图或 API 出图（同上）
4. 将图片插入文档对应位置

---

## 提示词编写规则

### 风格基准

每张图必须以 `templates/image-prompt-style.md` 定义的风格作为**唯一允许的基础风格**。

### 文案规则

- 图上的中文文字必须逐字指定，不允许模型自行添加
- 封面类/标题类图片：明确写"除指定文字外不要生成任何额外文字"
- 文字越少越好——能用图标/隐喻代替文字就不写文字

### 负面约束（必须包含）

- 不要扁平矢量海报风、不要 3D、不要摄影写实
- 不要复杂背景/强渐变光效
- 不要额外小字注解/水印/署名
- 不要英文与随机字符
